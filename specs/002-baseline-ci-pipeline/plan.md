# Implementation Plan: Baseline CI Pipeline (Pre-MVP Infrastructure)

**Branch**: `002-baseline-ci-pipeline` | **Date**: 2026-04-14 | **Spec**: [specs/002-baseline-ci-pipeline/spec.md](spec.md)

---

## Summary

Establish the GitHub Actions CI baseline workflow (`.github/workflows/ci.yml`) and enforce branch protection rules on all protected trunk branches (`main`, `master`, `develop`) **before any feature source-code development begins**. This is not a product feature - it is the foundational quality gate that every downstream epic depends on. The CI pipeline runs lint (`ESLint`), TypeScript type-check (`tsc --noEmit`), and unit/contract tests (`npm test`) on every pull request, with an optional conditional Terraform validation job. For early phases where frontend or IaC files are not scaffolded yet, the workflow uses capability detection and skips unavailable checks with clear notices instead of failing.

---

## Technical Context

**Language/Version**: YAML (GitHub Actions workflow syntax); TypeScript 5.x (type-check target)  
**Primary Dependencies**: GitHub Actions (`eyorg_windows_latest_8_32_A` runner - EY custom Windows); Node.js 20.x LTS; `hashicorp/setup-terraform@v3` (conditional); `actions/setup-node@v4`; `actions/cache@v4`  
**Storage**: N/A (no application data store for CI itself)  
**Testing**: The CI workflow *is* the test harness for the project - it runs Jest (`npm test`), ESLint (`npm run lint`), and `tsc --noEmit` against `src/frontend/`  
**Target Platform**: GitHub Actions (EY custom Windows runner `eyorg_windows_latest_8_32_A` - org-managed, Windows-aligned with local dev environment)  
**Project Type**: DevOps / CI pipeline configuration  
**Performance Goals**: CI workflow completes in under 10 minutes on baseline; under 15 minutes with full test suite (Next.js 14 app, Jest unit/contract tests)  
**Constraints**: Must run on GitHub-hosted runners; no privileged or self-hosted runner requirements; all secrets must be managed via GitHub's encrypted secret store; no Azure credentials needed for CI (only CD requires Azure auth); missing not-yet-developed modules (for example `src/frontend` or `infra/terraform`) must not hard-fail CI  
**Scale/Scope**: Single workflow file + branch protection UI configuration; 1 CI pipeline serving all 9 feature epics throughout the project lifetime

## Phase 1: Design & Contracts

> Detailed artifacts in [`data-model.md`](data-model.md), [`contracts/ci-workflow-contract.md`](contracts/ci-workflow-contract.md), [`quickstart.md`](quickstart.md).

### CI Workflow Structure (high-level)

```
on: pull_request (branches: main, master, develop)
jobs:
  lint         -> npm run lint         (src/frontend/, parallel)
  type-check   -> tsc --noEmit         (src/frontend/, parallel)
  test         -> npm test             (src/frontend/, parallel)
  terraform    -> fmt-check + validate (infra/terraform/, conditional, sequential after others)
```

### Branch Protection Configuration

| Setting | Value |
|---------|-------|
| Branches protected | `main`, `master`, `develop` |
| Required status checks | `lint`, `type-check`, `test` |
| No direct pushes | Enforced |
| Require PR review | 1 reviewer minimum |
| Dismiss stale reviews | On new commit push |
| Require up-to-date branch | Recommended |
